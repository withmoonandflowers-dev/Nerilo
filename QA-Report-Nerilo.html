1.4 失敗的測試案例詳細

所有 5 個失敗案例都在 P2PConnectionManager.spec.ts 的 ICE candidate buffering 區塊：

FAIL 1: ICE 先到、offer 後到 → ICE 先 buffer，offer 處理後 flush

FAIL 2: 多個 ICE 先到 → 全部 buffer，offer 後一次 flush

FAIL 3: ICE 先到、answer 後到（initiator 側）→ ICE buffer → answer 處理後 flush

FAIL 4: offer 已先到 → 後續 ICE 直接 addIceCandidate，不走 buffer

FAIL 5: 已有 remoteDescription 時重複 answer 應忽略

根因分析：測試 mock 中的 emitSignals() 呼叫 snapshot callback 後，handleSignal() 內的非同步 setRemoteDescription / addIceCandidate 未在 assertion 前完成。這很可能是 source code 中 signal processing 的 async/await 鏈有微妙的 race condition，導致 mock 環境下 Promise 未被正確 resolve。

2. 原始碼審查 — 發現的問題

CRITICAL 私鑰儲存在 localStorage

檔案：src/core/mesh/IdentityManager.ts (L100-152)

Ed25519 私鑰直接以 Base64 存入 localStorage，任何 XSS 漏洞都能竊取並冒充使用者身份。

建議：改用 IndexedDB 搭配加密儲存，或使用 Web Crypto API 的 non-extractable keys。

CRITICAL P2P Signal 處理中的 Race Condition

檔案：src/core/p2p/P2PConnectionManager.ts (L147-192)

remoteUid 在每次 handleSignal() 時被覆蓋。若多個 peer 同時發送信號，remoteUid 會被最後一個覆蓋，導致 signaling 混亂。此問題也是單元測試失敗的根本原因之一。

HIGH Signal 無速率限制

檔案：firestore.rules (signals collection)

任何已認證使用者都能無限制地在房間的 signals collection 中建立文件，可被利用進行 DoS 攻擊。

HIGH Gossip 訊息缺少新鮮度驗證

檔案：src/core/mesh/SecurityManager.ts (L51-88)

verifyMessage 只驗證密碼學簽章，不檢查訊息的時間戳。舊訊息可被重播 (replay attack)。

建議：加入 if (Date.now() - message.timestamp > MAX_AGE) return false;

HIGH meshIdentities 缺少輸入驗證

檔案：src/services/RoomService.ts (L718-758) + firestore.rules (L62-69)

userId 和 pubKey 在客戶端及 Firestore Rules 都只檢查 != null，未驗證格式、長度、Base64 合法性。惡意使用者可注入超大或畸形的 pubKey 造成記憶體問題。

MEDIUM SharedLedgerEngine Fork 偵測 Race Condition

檔案：src/core/ledger/SharedLedgerEngine.ts (L75-87)

detectFork() 和後續的 chain.find() 之間可能被並發 append() 修改，導致分支資訊不正確。

MEDIUM MeshTopologyManager 無重試機制

檔案：src/core/mesh/MeshTopologyManager.ts (L73-78)

鄰居連線失敗時只記錄 log，沒有重試或回退策略。在網路不穩定時可能導致 mesh 拓撲碎片化。

MEDIUM seenEntryHashes 清理可能遺漏

檔案：src/core/mesh/SharedDataStream.ts (L332-337)

清理策略保留後半部 hash，但可能丟棄仍在 entries 陣列中的 hash，導致重複 entry 被接受。

LOW P2PManager 未 await initializeServices()

檔案：src/core/p2p/P2PManager.ts (L64-77)

ondatachannel callback 中呼叫 initializeServices() 但未 await，服務初始化失敗會被靜默忽略。

LOW Firestore 資料轉換型別不安全

檔案：src/services/RoomService.ts (L163-172)

joinedAt 使用 fallback chain toMillis?.() || value || Date.now()，若原始值異常會默默替換成當前時間，掩蓋資料完整性問題。
