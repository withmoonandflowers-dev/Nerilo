## 🔧 測試問題與修復

### 當前問題

1. **連線狀態更新延遲**
   - **問題**：部分測試中連線狀態未及時更新為「已連線」
   - **原因**：重構後 hooks 的狀態更新邏輯需要調整
   - **狀態**：正在修復

2. **Mesh 測試超時**
   - **問題**：Mesh 測試需要更長的超時時間
   - **原因**：Mesh 架構需要建立多個連線，耗時較長
   - **狀態**：已調整超時時間到 90 秒

### 已修復的問題

1. ✅ **TypeScript 編譯錯誤** - 已修復
2. ✅ **架構選擇邏輯** - 已模組化
3. ✅ **訊息去重邏輯** - 已統一處理

## 📈 測試執行建議

### 快速測試（開發時）

```bash
# 只執行基本功能測試
npm run test:e2e -- tests/e2e/user-chat.spec.ts tests/e2e/room-management.spec.ts
```

### 完整測試（CI/CD）

```bash
# 執行所有測試
npm run test:e2e

# 並行執行（加速）
npm run test:e2e -- --workers=4
```

### 特定功能測試

```bash
# 測試架構選擇
npm run test:e2e -- tests/e2e/architecture-selection.spec.ts

# 測試 Mesh 架構
npm run test:e2e -- tests/e2e/mesh-gossip.spec.ts --timeout=180000
```
