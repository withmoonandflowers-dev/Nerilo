# ADR-0009：目標市場與產品形態

- 狀態：Accepted（2026-07-03 由產品負責人定調）
- 日期：2026-07-03

## Context

技術資產盤點顯示 Nerilo 的真實差異化是成本結構：訊息走 P2P 不經伺服器，
邊際成本趨近於零。原本評估三個方向：B2C 隱私聊天應用、嵌入式 SDK、
白牌垂直應用。

## Decision

產品負責人定調：**Nerilo 是資料傳遞架構，不是聊天產品**。

1. 首要目標是讓負責人未來開發的系統都能以 Nerilo 作為資料傳遞層，
   要求四大支柱：安全、正確、穩定、商業誘因（目標樹見
   [../GOAL-ANALYSIS.md](../GOAL-ANALYSIS.md)）。
2. 採「內部平台優先」路徑：先讓自有系統接入（dogfooding），
   再考慮 SDK 外部化（原選項 B 成為第二階段，按 MAU 計費的形態保留）。
3. 聊天 app 降格為參考應用：它是平台的第一個客戶、試煉場、
   以及收費機制（ADR-0008）的驗證場，不是終點產品。
4. 原選項 A（B2C 聊天主戰場）確定放棄；原選項 C（白牌）不主動經營，
   若有現成需求可作為現金流機會個案處理。

## Consequences

- ADR-0007 的休眠模組分類修訂：transport/（多通道 bus、store-and-forward）與 ordering/ 從「移除」改列「待啟用戰略資產」，因為它們是正確性支柱的實作載體。game/ 與 community/ 維持與定位無關，凍結或移除。
- 路線圖 M4 從「看市場再決定」升格為明確目的地：傳輸層抽取與平台契約（傳遞語義、加密、身分）文件化。
- ADR-0008 的計費架構不變（參考應用照收費），SDK 計費模型留待 M4 後定。
- 合規深度按內部平台標準起步，外部化前需補 SDK 等級的文件與相容性承諾。
