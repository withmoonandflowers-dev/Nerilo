# ADR-0011：中繼點數經濟（過路有對價）

- 狀態：Proposed（設計原則現在定，啟用與 GX 落地同步）
- 日期：2026-07-03

## Context

原始設計已內建「幫別人過路就得點數」的雛形：LocalCreditProvider
定義了 RelayReceipt（收發雙方簽章）、settleCredits() 結算介面、
防灌水節流與 free/basic/premium 分層（premium 門檻 1000 點），
原規劃第二階段以區塊鏈承載。ADR-0008 曾將其定位為「不作為計費依據、
保留介面」，理由是純 WebRTC 中繼的邊際成本近零，點數缺乏經濟實質。

ADR-0010 改變了前提。異質傳輸網路中，閘道節點承擔真實成本：
衛星橋接按則付費、LoRa 橋接消耗空中時間預算與電力、行動裝置
充當 mesh 中繼消耗電池。沒有對價機制，理性節點不會開放自己的
稀缺資源，GX 網路長不起來。同時，點數清算能替代「平台對每種通道
做計量帳單」的複雜度：網路內部自己算帳，平台只管兌換點。

已知的失敗模式（設計必須正面回答）：
- 點數偽造：目前結算端是本機記憶體，自己記自己的帳。
- Sybil：開分身互相「中繼」刷點。
- 自我中繼：流量繞過自己的另一個節點灌收據。
- 量測灌水：申報的中繼 KB 數無人驗證。
- 金流監理：點數若可提領現金，可能落入儲值支付或匯兌監理範圍。

## Decision

1. **帳本真相在伺服器端，不用區塊鏈**。settleCredits() 的實作改為
   Cloud Function：client 定期上繳簽章收據，伺服器驗簽、去重、
   記入 Firestore 帳本。原「第二階段區塊鏈」擱置：目前規模下，
   中心化帳本的信任假設（使用者本來就信任平台的計費）已足夠，
   區塊鏈只增加成本不增加信任。IIncentiveProvider 介面不變，
   保留未來抽換空間。
2. **收據可信度靠三方簽章結構**：中繼收據需要「發送端簽發、
   中繼端背書、接收端確認送達」三段才算有效。單方或雙方收據
   只記錄不計點。這使自我中繼與量測灌水需要合謀三個身分。
3. **Sybil 防線與帳號成本掛鉤**：匿名與新帳號的點數累積上限極低；
   通過付費驗證（Pro 訂閱）或長期信譽（PeerScoring 分數）的帳號
   才解鎖完整累積速率。搭配既有的每小時累積上限。
4. **兌現只有一條路：折抵訂閱費**。點數不可提領現金、不可轉讓。
   這把系統定性為「回饋折抵」而非「儲值支付」，避開金流監理，
   也消滅點數交易市場的投機動機。閘道節點營運者（真實成本最高者）
   的對價形態是「訂閱全免加額度」，若未來出現專業閘道營運需求，
   再另訂商業合約，不走點數提領。
5. **計價分級跟著 ADR-0010 的通道等級走**：broadband 中繼低費率
   （近象徵性）、narrowband 中繼中費率（空中時間稀缺）、
   message 通道（衛星）高費率且按則計價。費率表集中設定，
   與 ADR-0005 配額表同一機制管理。
6. **受益者均攤（2026-07-03 增補）**：中繼成本不由發送者單獨負擔，
   由該資料流的受益方分攤。具體為房間層級清算：一則訊息經中繼投遞，
   點數費用按房間成員數均攤記帳（發送者與接收者都是受益人）；
   成員自己也在貢獻中繼時，同一本帳上先互抵再結餘。
   兩個配套：
   - 房主代付模式：房主（或 Pro 訂閱者）可宣告承擔全房費用，
     支撐「邀請朋友零門檻」的獲客情境。
   - 新帳號免費額度：小額起始點數，用完前不會因欠費被拒投遞；
     欠費帳號降級為僅直連（P2P 免費路徑不受影響），不是斷線。
   這使「負擔共享、付費也共享」成為帳本的預設語義，
   而非事後補丁；也讓多路徑分流（GT6）的成本歸屬有清楚答案：
   幾條路徑就幾份中繼收據，全部進同一次均攤。
7. **啟用時點**：M4 只做帳本結算端與收據驗證（基礎設施），
   點數對使用者可見但不可消費（累積期，同時收集詐欺樣態）；
   M5 GX 通道上線時開放折抵，經濟閉環啟動。

## Consequences

- 「提供過路就有點數」從設計文件變成有結算保障的承諾，且法律風險被兌現路徑的選擇（僅折抵）控制在最小。
- 放棄區塊鏈敘事，投資人簡報少一個 buzzword，多一個「我們知道什麼時候不用它」的工程判斷。
- 三方簽章收據增加協議複雜度與 payload 開銷，窄頻通道上收據本身也占空中時間，需要批次與壓縮設計（M5 工程細節）。
- 伺服器端帳本寫入是新的 Firestore 成本項，批次結算（例如每小時一次）控制在低量級。
- PeerScoring 從單純的路由品質信號升格為信譽資產，其防操縱設計（GossipSub v1.1 模型）需要一次針對「刷信譽」的複審。
- 點數不可轉讓的決定若未來要翻案（例如開放節點間交易），必須重新過金流監理評估，屆時開新 ADR 取代本篇。
