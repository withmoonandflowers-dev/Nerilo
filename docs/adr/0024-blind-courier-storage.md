# ADR-0024：盲信使儲存經濟學——配額、刪除、完整性

- 狀態：Accepted（設計決策；實作屬 ADR-0023 P4）
- 日期：2026-07-06
- 承接：ADR-0023 修訂二 R1（盲信使）、ADR-0021/0022（中繼即價值、共簽收據）

## Context

盲信使 = 非成員 C 為他人房間保存密文複本並參與補齊。產品負責人三問：
①一般人要幫別人存多少？②怎麼刪？③要不要改存 hash 防竄改？

## Decision 1：存「完整密文紀錄」，不是 hash

- **防竄改已由簽章解決**：每筆紀錄帶寄件人 ECDSA 簽章（涵蓋 content、seq、
  messageId、channel）。信使改任何一個 byte，收端驗簽失敗即丟。✅ 現行機制。
- **只存 hash = 自廢武功**：hash 不可逆——成員回線向信使要資料時，
  hash 給不出任何東西，盲信使的存在意義（補齊）就沒了。
- **結論**：竄改「會被嘗試」但「不可能得逞」，不造成困擾；hash 方案不採。

### 誠實的殘餘威脅：寄件人分叉（equivocation）

簽章擋第三方，擋不了**寄件人自己**對同一 (senderId, seq) 簽兩份不同內容、
發給不同節點——兩份都是真簽章。現況 first-write-wins，節點間可能短暫分歧。
- 影響評估：需要寄件人惡意 + 刻意分流，且對方拿不到好處（聊天語境）；
  ADR-G01 信任模型（朋友間）下非阻塞。
- 防禦路線（P5 審計強化）：紀錄鏈上前筆 hash（休眠 `chain/` 模組即為此設計），
  分叉出現即全網可證。屆時信使儲存格式不變（鏈欄位在紀錄內、隨簽章走）。

## Decision 2：配額——信使是快取，不是檔案庫

**權威複本永遠在成員本地（P1）**；信使只橋接成員離線的間隙。
推論：信使丟資料只降可用性、不損正確性 → 敢用小預算 + 激進淘汰。

| 參數 | 預設 | 依據 |
|---|---|---|
| 單筆紀錄上限 | 4 KB | 文字紀錄 ~1KB（密文+簽章+信封）；對齊固定封包設計。檔案不代存（Phase 1） |
| 單房上限 | 5 MB | ≈ 活躍房 50 天文字量（100 則/天 ≈ 100KB/天） |
| 信使總預算 | 100 MB（可調 0/50/100/500） | ≈ 20 房 × 7 天視窗 × 7 倍餘裕；瀏覽器單站配額 GB 級，手機無感 |
| 保存視窗 | 14 天 TTL | 成員離線超過兩週屬罕見；更久的補齊回退成員互補 |

## Decision 3：刪除——四層機制

1. **TTL 過期**：寄存時標註時間，14 天自動過期清除。
2. **預算 LRU**：總量觸頂時，淘汰最久未被存取的房。
3. **簽章墓碑**：房間真刪／成員退出時，成員以房籍身分簽署 tombstone
   廣播給信使；信使驗章即刪（盲的也能執行——墓碑驗證只需 pubKey 與房籍證明）。
4. **使用者主權**：手動清空 + 總開關（預設參與、設定頁可關；文案誠實
   標示「開著頁面才在幫忙」）。

## Consequences

- 信使成本低到可以「預設開啟」而不造成使用者負擔 → 網絡效應的冷啟動阻力最小化。
- 快取定位使複本安全性分析簡單：最壞情況 = 沒有信使 = 回到 P1 現況（成員互補）。
- 檔案/媒體代存（大體積）刻意排除在 Phase 1 外——那是另一個經濟學（需配額市場
  或付費點數），不要讓它拖垮文字訊息的網絡啟動。
- 計量與激勵沿用 ADR-0022 共簽收據 → 點數；PeerScoring 對「收了不存/不服務」
  的信使降分（多副本 K=3 起，容忍單信使擺爛）。
