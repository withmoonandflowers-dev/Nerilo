# ADR-0029：信使經濟採有對象、可同意交換的欠條

- 狀態：Accepted（Spec 001 已落地）
- 日期：2026-07-16
- 關聯：Spec 001、Protocol Spec 003、ADR-0011、ADR-0024

## 背景

盲信使提供稀缺儲存與頻寬，免費寄存會讓單一身分以近零成本耗盡快取；但把服務量抽象成可自由流通的 coin 或全網餘額，會引入全域發行、共識、清算與監理問題。系統需要限制垃圾寄存，同時保留沒有信使也能運作的免費成員互補路徑。

## 決策

信使經濟的計算單位是有明確對象的欠條，不是 coin：

1. 每個信使維護獨立債權簿，只判斷自己目前持有的欠條；不存在全網餘額或同質代幣。
2. 寄存者依該信使的限時報價，向該信使簽發 `IOU(寄存者→信使, 金額)`。每個發票人在每個信使有獨立未結上限 ε。
3. 服務貢獻形成 `IOU(受益人→服務者, 金額)`。不同發票人的欠條不等價；只有原發票人、目前持有人與新持有人都簽署時，欠條才移轉。
4. 信使接受第三方服務欠條後，可把寄存者自己開的欠條退回。只有該寄存者本人簽署結清授權，債權簿才註銷欠條並恢復額度。
5. 報價由各信使按本地利用率自主決定；協議只固定報價、簽章、交換、結清與拒收語義。
6. 信使是可拒絕的加速／離線代管層。信使滿載、拒收或不存在時，不得刪除成員本地權威紀錄；成員間 anti-entropy 仍免費運作。

## 取捨

- 選有對象欠條而非 coin：避免鑄造、全域共識與統一清算，但流動性被分割；接收者可拒絕某發票人的欠條，交換價格也由當事人自行同意。
- 選每信使獨立 ε：攻擊成本會隨使用的信使數增加，也讓信使能自主承擔風險；代價是沒有單一「可用餘額」可展示。
- 選三方同意轉讓：保留發票人對債權人變更的同意權，代價是發票人離線時交換不能完成。
- 選本人結清：防止信使或第三方單方面抹除債務；代價是結清需要發票人的可用私鑰。

## 後果與邊界

- 寄存垃圾會耗盡攻擊者對該信使的 ε，但不會把免費 P2P 底線變成付費牆。
- 欠條簿仍由單一信使本地權威維護；同一瀏覽器的 IndexedDB 快照與逐筆驗章復原已由 ADR-0030／Spec 004 補上。跨裝置備份、私鑰復原與多裝置合併仍未定義。
- Protocol Spec 003 固定 v1 線上格式。任何改變簽章欄位或 canonical 序列化的修改，都是協議破壞性變更。
- 本 ADR 取代 ADR-0011 對「全域點數不可轉讓」在盲信使寄存場景的假設；ADR-0011 其他商業回饋帳本決策不受影響。
