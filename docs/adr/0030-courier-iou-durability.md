# ADR-0030：信使欠條簿採本機耐久快照與重載重驗

- 狀態：Accepted（2026-07-16 已落地）
- 日期：2026-07-16
- 關聯：ADR-0029、Protocol Spec 003、Spec 004

## 背景

每個信使的債權簿是本地權威。若只存在記憶體，頁面重載會同時遺失未結債權、已接受服務收據與防重放狀態，讓同一 issuer 的額度被重置，也可能重複接受已交換收據。

## 決策

1. Vue 信使以 `ownerNodeId` 為鍵，在獨立 IndexedDB 保存一列完整 JSON 快照；單列 replace 讓欠條註銷與服務欠條轉入一起提交。
2. 快照保存寄存欠條及 issuer 公鑰、已接受的貢獻欠條、防重放 nonce 與定價狀態。短命、單次使用的報價不跨重載保存。
3. 重載時不信任本機資料：逐張重驗 nodeId／公鑰綁定、簽章、holder 與 per-issuer 上限；損壞或超額項目跳過，再覆寫乾淨快照。
4. 已配置耐久層時，寄存必須等密文紀錄與欠條快照都落定才回成功 ACK。任一寫入失敗時補償回滾兩邊並回 `persistence-failed`。
5. 結清亦須先完成原子快照寫入才成功；失敗時恢復原寄存債權與防重放狀態。

## 取捨與邊界

- 完整快照簡化原子性與損壞清理，適合目前每信使受儲存預算限制的小型債權簿；資料量放大後可改事件日誌，但不得降低 ACK 後耐久保證。
- IndexedDB 解決同一瀏覽器重載與程序重啟，不等於備份。跨裝置匯出必須連同信使私鑰復原、加密與衝突合併一起設計，不能只複製 JSON。
- 瀏覽器不支援 IndexedDB 時維持既有記憶體模式；該節點不宣稱跨重載耐久。
