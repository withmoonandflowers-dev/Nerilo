# Nerilo：斷網也能送達的加密通訊（一頁提案）

## 問題

台灣的日常通訊幾乎全押在集中式服務上。地震、颱風、海纜受損、基地台過載，任何一環出事，
LINE 和 Messenger 就是一起斷。對防災單位、偏鄉社區、需要隱私保障的族群（記者、社工、
受暴者支援網絡）來說，「平時方便」的代價是「關鍵時刻不可用，且平台看得到一切」。

## 方案

Nerilo 是一套瀏覽器就能跑的點對點加密通訊技術。三個核心能力：

1. **不經伺服器的加密通訊**：訊息走裝置對裝置的 WebRTC 直連，內容以端到端加密保護，
   伺服器只協助牽線，看不到任何明文。
2. **斷線也送得到（盲信使）**：收件者離線時，訊息以密文寄放在路過的陌生節點上，對方
   回線後自動領取。信使全程只見密文，驗得了真偽、讀不了內容，送達後憑雙方共簽的收據
   賺取點數，形成互助循環。
3. **別人可以直接拿去用（SDK）**：整套能力封裝成可嵌入的開源 SDK，其他公益專案（社區
   防災 App、匿名通報系統）不需要理解密碼學細節，帶自己的後端即可串接。

## 為什麼可信（可查核的技術主張）

- 訊息「恰好一次」送達：寄收兩端不掉訊、不重複，有自動化端對端測試持續驗證。
- 端到端加密：伺服器與信使僅見密文，經第三方視角的核心不變量稽核（報告隨附，
  結論為有條件通過，已知風險全數自我揭露並排入處置）。
- 品質制度：1,300 餘項單元測試、模擬多人情境的自動化瀏覽器測試、CI 硬性閘門，
  每次改動都須通過才能合併。
- 全部開源，稽核報告與架構決策紀錄（25 篇 ADR）公開可驗。

## 社會影響

- **防災韌性**：基礎設施受損時，社區內的裝置互為信使，訊息仍能輾轉送達。
- **隱私人權**：通訊內容不落任何第三方之手，適合高風險溝通場景。
- **公共財**：以 SDK 形式開放，讓影響力不限於單一 App，而是一層可被重複使用的基礎能力。

## 現況與需求

現況：可操作的網頁版 demo（三分鐘可演示斷線送達全流程）、可發佈的 SDK、完整稽核文件。
本專案為個人與 AI 協作開發，申請資源將用於：TURN 中繼與測試基礎設施費用、多裝置實地
測試（偏鄉或演習場景）、SDK 文件與範例的在地化整理。
