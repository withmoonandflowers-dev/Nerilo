# 上板與部署手冊

本專案以 **Firebase Hosting** 為前端上板目標，並可選部署 Firestore 規則與 Cloud Functions。以下為標準上板指令、流程與注意事項。

---

## 0. 前置與環境（首次設定）

### 0.1 環境需求

- Node.js 18+
- npm 或 yarn
- Firebase CLI（`npm install -g firebase-tools`、`firebase login`）
- Firebase 專案已建立，並以 `firebase init` 選好 Firestore、Functions、Hosting

### 0.2 環境變數

複製 `env.example.txt` 為 `.env.local`，填入 Firebase 設定（勿提交 `.env.local`）：

- `VITE_FIREBASE_API_KEY`、`VITE_FIREBASE_AUTH_DOMAIN`、`VITE_FIREBASE_PROJECT_ID`、`VITE_FIREBASE_STORAGE_BUCKET`、`VITE_FIREBASE_MESSAGING_SENDER_ID`、`VITE_FIREBASE_APP_ID`

Cloud Functions 若需 Twilio：`firebase functions:config:set twilio.account_sid="..." twilio.auth_token="..."`

### 0.3 首次部署 Firestore / Functions

- 規則與索引：`firebase deploy --only firestore:rules`、`firebase deploy --only firestore:indexes`
- Functions：`cd functions && npm install && npm run build`，再 `firebase deploy --only functions`
- features 等初始資料請於 Firebase Console 手動建立（見專案內說明）。

## 1. 標準上板指令

### 1.1 僅部署 Hosting（最常用）

前端建置後只部署靜態站台到 Firebase Hosting：

```bash
# 等同：npm run build && firebase deploy --only hosting
npm run deploy
```

或直接使用 Firebase CLI：

```bash
firebase deploy --only hosting
```

**注意**：`firebase deploy --only hosting` 會使用目前 `dist/` 目錄內容；若未先建置，會部署到舊版。建議一律先執行 `npm run build` 或使用 `npm run deploy`。

### 1.2 僅 Hosting（npm 別名）

```bash
npm run deploy:hosting
```

與 `npm run deploy` 相同：先 build，再 `firebase deploy --only hosting`。

### 1.3 部署全部（Hosting + Firestore + Functions）

```bash
npm run deploy:full
```

會先建置前端，再執行 `firebase deploy`（hosting、firestore rules/indexes、functions 一併部署）。

### 1.4 上板前檢查（建議正式上板使用）

先執行型別檢查、Lint、單元測試，通過後再建置並部署：

```bash
npm run deploy:safe
```

或使用腳本（PowerShell）：

```powershell
.\scripts\deploy.ps1 -Check
```

---

## 2. 腳本一覽（PowerShell）

| 指令 | 說明 |
|------|------|
| `.\scripts\deploy.ps1` | 建置 + `firebase deploy --only hosting` |
| `.\scripts\deploy.ps1 -Check` | 先 type-check、lint、test:run，通過後再 build + deploy hosting |
| `.\scripts\deploy.ps1 -Full` | 建置 + `firebase deploy`（全部） |
| `.\scripts\deploy-safe.ps1` | 等同 `deploy.ps1 -Check`，一鍵安全上板 |

腳本會自動辨識專案根目錄（含 `package.json` 的目錄），無需先 `cd` 到專案根。

## 3. 環境需求

- **Node.js**：建議 v18+
- **npm**：已執行 `npm install`
- **Firebase CLI**：已安裝並登入  
  - 安裝：`npm install -g firebase-tools`  
  - 登入：`firebase login`
- **專案綁定**：`.firebaserc` 已設定目標專案（如 `nerilo`）

建置產物輸出至 `dist/`，與 `firebase.json` 中 `hosting.public` 一致。

---

## 4. 上板流程建議（專業流程）

### 4.1 日常／快速上板（僅改前端）

1. 確認本地程式已存檔、無編譯錯誤。
2. 執行：
   ```bash
   npm run deploy
   ```
3. 到 Firebase Console 或 Hosting URL 確認版本是否更新。

### 4.2 正式／釋出上板（建議跑檢查）

1. 執行檢查與部署：
   ```powershell
   .\scripts\deploy.ps1 -Check
   ```
   或手動：
   ```bash
   npm run type-check && npm run lint && npm run test:run && npm run deploy
   ```
2. 若需一併更新 Firestore 規則或 Functions：
   ```powershell
   .\scripts\deploy.ps1 -Check -Full
   ```
3. 部署完成後，依需求做 smoke test（開啟 Hosting URL、登入、建立房間、發送訊息等）。

### 4.3 CI/CD 上板（例如 GitHub Actions）

- 在 CI 中依序執行：`npm ci` → `npm run type-check` → `npm run lint` → `npm run test:run` → `npm run build` → `firebase deploy --only hosting --token "$FIREBASE_TOKEN"`。
- 可參考專案內 `.github/workflows/` 既有設定，將「建置 + 部署」接在測試通過之後。

---

## 5. 與 firebase.json 對應

- **hosting**：`public` 為 `dist`，所有路徑 rewrite 至 `/index.html`（SPA）。
- **functions**：若有部署，會先執行 `npm --prefix functions run build`（predeploy）。
- **firestore**：規則檔 `firestore.rules`、索引 `firestore.indexes.json`。

僅執行 `firebase deploy --only hosting` 時，不會動到 Firestore 與 Functions。

---

## 6. 常見問題

- **部署後仍是舊版**：確認有先執行 `npm run build`，且部署的是同專案、同站點。
- **build 失敗**：先執行 `npm run type-check` 與 `npm run lint` 排查錯誤。
- **firebase: command not found**：請安裝並將 Firebase CLI 加入 PATH：`npm install -g firebase-tools`。
- **權限錯誤**：使用 `firebase login` 重新登入，或使用 CI 的 `FIREBASE_TOKEN`。

---

## 7. 舉一反三：其他常用指令

| 需求 | 指令 |
|------|------|
| 僅建置不部署 | `npm run build` |
| 本地預覽建置結果 | `npm run preview` |
| 僅部署 Firestore 規則 | `firebase deploy --only firestore:rules` |
| 僅部署 Functions | `firebase deploy --only functions` |
| 指定專案部署 | `firebase use <project-id>` 後再 `npm run deploy` |

以上搭配 `npm run deploy` / `firebase deploy --only hosting` 即可涵蓋日常上板與進階部署需求。
