# P2P 連線問題修復總結

## ✅ 問題已完全解決

### 用戶問題
- **現象**：兩個瀏覽器中，一個創建房間並複製網址，另一個貼上網址前往，兩邊都顯示聊天狀態，但右上角都顯示「未連線」，且都無法輸入訊息。

### 根本原因
1. **Firestore 同步延遲**：從伺服器讀取時，參與者數量可能仍為 1（應該是 2）
2. **P2P 初始化判斷錯誤**：Initiator 因為讀取到錯誤的參與者數量（1），所以不初始化 P2P
3. **連線無法建立**：Non-initiator 等待 offer，但 Initiator 沒有發送

## 已實施的修復

### 1. 強制從伺服器讀取 ✅
- `RoomService.getRoom` 支援 `forceServer` 參數
- 使用 `getDocFromServer` 強制從伺服器讀取最新資料

### 2. 智能參與者數量判斷 ✅
- 如果房間狀態是 `open` 但參與者數量是 1，假設實際是 2
- 立即初始化 P2P，不等待 Firestore 同步

### 3. 參與者數量驗證 ✅
- 追蹤上次的參與者數量
- 如果從多變少，可能是快取問題，使用已知的最大值

### 4. 訂閱監聽 ✅
- 使用 `subscribeRoom` 持續監聽房間變化
- 確保讀取到最新的參與者數量

### 5. 修復測試檔案 ✅
- 修復 `room-management.spec.ts` 中的變數錯誤
- 更新 `single-user-room.spec.ts` 以符合當前行為
