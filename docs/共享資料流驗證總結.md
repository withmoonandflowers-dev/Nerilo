# 共享資料流：效能與安全驗證總結

## 驗證範圍

針對 **SharedDataStream** 與 **crypto** 工具進行：

1. **安全性強化**：輸入驗證、速率限制、大小限制、型別與格式檢查  
2. **效能保護**：Payload 與 hash 輸入上限、條目數與 reset 上限、seen set 清理  
3. **完整單元測試**：36 個測試涵蓋正常與異常路徑  

## 已實施措施

### 1. crypto（`src/utils/crypto.ts`）

| 項目 | 說明 |
|------|------|
| `sha256Hash` | 僅接受 string；輸入長度上限 1,000,000 字元，超過拋 `RangeError` |
| `isPlainObject` | 僅接受純物件，拒絕 null、陣列、非 object，避免 __proto__ 污染 |
| `isHex64` | 驗證 64 字元 hex（SHA-256 輸出格式） |
| `computePayloadHash` | payload 須為純物件；可傳入 `maxSerializedLength`（預設 100KB） |
| `computeEntryHash` | 驗證 index 非負整數、timestamp 非負整數、creatorId 非空、payloadHash 為 64 字元 hex |

### 2. SharedDataStream（`src/core/mesh/SharedDataStream.ts`）

| 項目 | 說明 |
|------|------|
| **append** | payload 須為純物件；檢查 `maxEntries`、`appendRateLimitPerSecond`（預設 20/秒）；payload 透過 `computePayloadHash` 受 `maxPayloadSize` 限制 |
| **handleReceivedEntry** | 先驗證 entry 結構（`validateEntryShape`）：index、timestamp、creatorId、payload 純物件、entryHash/payloadHash 為 64 hex；再驗證 hash 鏈、index 連續、payloadHash；達 `maxEntries` 或 payload 過大則拒絕 |
| **resetFromEntries** | 僅接受陣列且長度 > 0；遠端條目數上限 100,000；不得超過 `maxEntries`；每筆皆經 `validateEntryShape` 與 hash/payloadHash 驗證 |
| **config** | `maxPayloadSize`（預設 100KB）、`maxEntries`（預設 50,000）、`appendRateLimitPerSecond`（預設 20）、`hashHexLength`（64） |

### 3. 型別（`src/types/index.ts`）

- `SharedStreamConfig` 新增：`maxPayloadSize`、`maxEntries`、`appendRateLimitPerSecond`、`hashHexLength`（皆可選）。

## 測試結果

- **執行指令**：`npm run test:run`（Vitest）
- **結果**：2 個測試檔、36 個測試全部通過
  - `tests/unit/crypto.spec.ts`：19 個測試（sha256Hash、isPlainObject、isHex64、computePayloadHash、computeEntryHash）
  - `tests/unit/SharedDataStream.spec.ts`：17 個測試（append、handleReceivedEntry、resetFromEntries、getEntries/getLastEntry、監聽與廣播）
- **Type-check**：`npm run type-check` 通過
- **Lint**：相關檔案無 linter 錯誤

## 建議後續

- 若需與 Mesh Gossip 整合：在 Gossip 訊息中承載 LedgerEntry（例如 `type: 'ledger_entry'`），並將 `setBroadcastHandler` 接到鄰居的 `send`，收到時呼叫 `handleReceivedEntry`。
- 可依需求調整 `maxPayloadSize`、`maxEntries`、`appendRateLimitPerSecond`（透過 `SharedStreamConfig`）。

## 總結

共享資料流已具備：

- **效能**：Payload 與 hash 輸入上限、條目數與 reset 上限、seen set 清理，避免記憶體與 CPU 濫用。  
- **安全**：純物件與 64 hex 驗證、append 速率限制、hash 鏈與 payloadHash 驗證，降低竄改與濫用風險。  
- **驗證**：36 個單元測試覆蓋正常與異常路徑，type-check 與 lint 通過。
