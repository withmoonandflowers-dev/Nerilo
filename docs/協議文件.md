# P2P 傳輸協議文件

## 1. 協議概述

Nerilo 使用統一的 Envelope 格式進行所有 P2P 資料傳輸（除檔案二進位資料外）。此協議設計確保：

- 可擴充性：新功能只需定義 namespace/schema
- 類型安全：TypeScript 類型定義
- 驗證機制：P2PProtocolRegistry 驗證

## 2. Envelope 格式

### 2.1 基本結構

```typescript
interface P2PEnvelope {
  v: number;              // 協議版本（目前為 1）
  ns: string;              // Namespace（見下方）
  type: string;            // 訊息類型
  id: string;              // UUID（唯一識別）
  ts: number;              // 時間戳（毫秒）
  from: string;            // 發送者（格式：uid/deviceId）
  to?: string;             // 接收者（可選，格式：uid/deviceId）
  replyTo?: string;        // 回覆的訊息 ID（可選）
  payload: any;            // 訊息內容
  meta?: Record<string, any>; // 元資料（可選）
}
```

### 2.2 欄位說明

- **v**：協議版本，用於未來相容性處理
- **ns**：Namespace，用於路由訊息到對應 handler
- **type**：訊息類型，在 namespace 內唯一
- **id**：UUID，用於去重、追蹤
- **ts**：時間戳，用於排序、衝突解決
- **from**：發送者識別（uid/deviceId 格式支援跨裝置同步）
- **to**：接收者識別（可選，用於點對點訊息）
- **replyTo**：回覆的訊息 ID（可選）
- **payload**：訊息內容，格式依 namespace/type 而定
- **meta**：元資料（可選），用於擴充資訊

## 3. 官方保留 Namespace

### 3.1 system

系統層級訊息，用於錯誤處理、心跳等。

**Types**：
- `ERROR`：錯誤訊息
- `PING`：心跳請求
- `PONG`：心跳回應
- `ACK`：確認收到

**ERROR Payload**：
```typescript
{
  namespace: string;
  type: string;
  message: string;
}
```

### 3.2 chat

文字聊天訊息。

**Types**：
- `MSG_SEND`：發送訊息
- `MSG_EDIT`：編輯訊息
- `MSG_DELETE`：刪除訊息
- `TYPING`：輸入中狀態

**MSG_SEND Payload**：
```typescript
{
  messageId: string;
  from: string;
  to?: string;
  content: string;
  timestamp: number;
  edited?: boolean;
  deleted?: boolean;
}
```

**MSG_EDIT Payload**：
```typescript
{
  messageId: string;
  content: string;
}
```

**MSG_DELETE Payload**：
```typescript
{
  messageId: string;
}
```

**TYPING Payload**：
```typescript
{
  isTyping: boolean;
}
```

### 3.3 file

檔案傳輸。

**Types**：
- `FILE_META`：檔案 metadata
- `FILE_CHUNK`：檔案分片
- `FILE_END`：傳輸完成
- `FILE_CANCEL`：取消傳輸

**FILE_META Payload**：
```typescript
{
  fileId: string;
  fileName: string;
  fileSize: number;
  fileType: string;
  chunkCount: number;
  chunkSize: number;
}
```

**FILE_CHUNK Payload**：
```typescript
{
  fileId: string;
  chunkIndex: number;
  data: string; // Base64 編碼
}
```

**FILE_END Payload**：
```typescript
{
  fileId: string;
}
```

**FILE_CANCEL Payload**：
```typescript
{
  fileId: string;
}
```

### 3.4 media

媒體狀態同步（實際媒體走 WebRTC tracks）。

**Types**：
- `MEDIA_READY`：媒體已準備
- `MEDIA_TOGGLE`：媒體狀態變更
- `MEDIA_END`：結束媒體

**MEDIA_READY Payload**：
```typescript
{
  audioEnabled: boolean;
  videoEnabled: boolean;
}
```

**MEDIA_TOGGLE Payload**：
```typescript
{
  audioMuted?: boolean;
  videoMuted?: boolean;
}
```

**MEDIA_END Payload**：
```typescript
{}
```

### 3.5 sync

跨裝置同步。

**Types**：
- `INDEX`：發送訊息索引
- `REQ`：請求訊息
- `RES`：回應訊息
- `DONE`：同步完成

**INDEX Payload**：
```typescript
{
  index: Array<{
    messageId: string;
    deviceId: string;
    timestamp: number;
    hash: string;
  }>;
}
```

**REQ Payload**：
```typescript
{
  messageIds: string[];
}
```

**RES Payload**：
```typescript
{
  messages: ChatMessage[];
}
```

**DONE Payload**：
```typescript
{}
```

## 4. 功能自訂 Namespace

### 4.1 命名規範

格式：`feature.<name>`

範例：
- `feature.crm`
- `feature.analytics`
- `feature.custom`

### 4.2 註冊流程

```typescript
const registry = new P2PProtocolRegistry();

registry.register({
  namespace: 'feature.crm',
  types: ['CONTACT_UPDATE', 'DEAL_CREATE', 'NOTE_ADD'],
  payloadSchema: {
    CONTACT_UPDATE: {
      contactId: 'string',
      fields: 'object',
    },
  },
  validator: (payload) => {
    // 自訂驗證邏輯
    return payload.contactId && typeof payload.contactId === 'string';
  },
}, async (envelope) => {
  // Handler 邏輯
  console.log('Received CRM message:', envelope);
});
```

### 4.3 驗證規則

1. Namespace 必須以 `feature.` 開頭
2. 必須定義至少一個 type
3. 可選：定義 payloadSchema
4. 可選：提供 validator 函數

## 5. 協議驗證流程

### 5.1 基本驗證

P2PProtocolRegistry 會檢查：

1. 必填欄位：v, ns, type, id, ts, from, payload
2. Namespace 是否註冊（非保留 namespace）
3. Type 是否在允許的 types 中
4. Payload 是否符合 schema（如有定義）

### 5.2 錯誤處理

驗證失敗時，P2PChannelBus 會發送 `system.ERROR`：

```typescript
{
  v: 1,
  ns: 'system',
  type: 'ERROR',
  id: 'error_xxx',
  ts: Date.now(),
  from: 'system',
  payload: {
    namespace: 'feature.unknown',
    type: 'INVALID_TYPE',
    message: 'Unregistered namespace: feature.unknown',
  },
}
```

## 6. 使用範例

### 6.1 發送聊天訊息

```typescript
const envelope: P2PEnvelope = {
  v: 1,
  ns: 'chat',
  type: 'MSG_SEND',
  id: generateUUID(),
  ts: Date.now(),
  from: `${uid}/${deviceId}`,
  payload: {
    messageId: generateUUID(),
    from: `${uid}/${deviceId}`,
    content: 'Hello, World!',
    timestamp: Date.now(),
  },
};

await channelBus.send(envelope);
```

### 6.2 訂閱訊息

```typescript
channelBus.subscribe('chat', async (envelope) => {
  if (envelope.type === 'MSG_SEND') {
    const message = envelope.payload as ChatMessage;
    // 處理訊息
    await saveToIndexedDB(message);
    updateUI(message);
  }
});
```

### 6.3 註冊自訂功能

```typescript
// 定義協議
registry.register({
  namespace: 'feature.analytics',
  types: ['EVENT_TRACK', 'PAGE_VIEW'],
  validator: (payload) => {
    return payload.event && typeof payload.event === 'string';
  },
}, async (envelope) => {
  if (envelope.type === 'EVENT_TRACK') {
    // 處理分析事件
    trackEvent(envelope.payload);
  }
});

// 發送訊息
const envelope: P2PEnvelope = {
  v: 1,
  ns: 'feature.analytics',
  type: 'EVENT_TRACK',
  id: generateUUID(),
  ts: Date.now(),
  from: `${uid}/${deviceId}`,
  payload: {
    event: 'button_click',
    properties: { buttonId: 'submit' },
  },
};

await channelBus.send(envelope);
```

## 7. 版本相容性

目前協議版本為 1。未來如需變更：

1. 增加版本號（v: 2）
2. 保持向後相容（支援 v: 1）
3. 在 P2PChannelBus 中處理版本差異

## 8. 最佳實踐

1. **總是使用 UUID**：確保訊息唯一性
2. **包含時間戳**：用於排序、衝突解決
3. **使用 from/to 格式**：`uid/deviceId` 支援跨裝置同步
4. **驗證 payload**：使用 validator 確保資料正確性
5. **錯誤處理**：監聽 `system.ERROR` 並適當處理
