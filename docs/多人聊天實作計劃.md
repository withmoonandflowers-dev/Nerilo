# 多人聊天實作計劃

## 架構設計

### 星型拓撲（Star Topology）

```
        [房主/Host]
            /  |  \
           /   |   \
    [User1] [User2] [User3]
```

- 房主與每個用戶建立獨立的 P2P 連線
- 每個連線使用獨立的 DataChannel（label: `chat-{remoteUid}`）
- 房主負責轉發訊息給所有其他用戶

## 實作步驟

### 階段 1：基礎架構 ✅

1. ✅ 建立 `MultiP2PManager` 類別
2. ✅ 實作連線管理（addConnection, removeConnection）
3. ✅ 實作訊息路由（broadcastMessage, forwardMessage）

### 階段 2：整合到 ChatPage

1. **修改 ChatPage 支援多人**：
   - 檢測參與者數量
   - 如果 > 2，使用 MultiP2PManager
   - 如果 = 2，使用現有的 P2PManager（向後兼容）

2. **處理新用戶加入**：
   - 房主檢測到新參與者時，建立新連線
   - 新用戶檢測到房主時，建立連線

3. **處理用戶離開**：
   - 監聽參與者列表變化
   - 自動清理斷開的連線

### 階段 3：訊息處理

1. **房主邏輯**：
   - 收到訊息後，轉發給其他所有用戶
   - 自己發送的訊息，廣播給所有用戶

2. **非房主邏輯**：
   - 只與房主建立連線
   - 所有訊息都通過房主轉發

### 階段 4：連線狀態管理

1. **監控每個連線的狀態**
2. **顯示整體連線狀態**（例如：3/5 已連線）
3. **自動重連失敗的連線**

## 實作細節

### MultiP2PManager 使用方式

```typescript
// 初始化
const multiP2P = new MultiP2PManager(roomId, localUid, isHost);

// 建立連線（房主專用）
if (isHost) {
  for (const participantUid of participants) {
    if (participantUid !== localUid) {
      await multiP2P.addConnection(participantUid);
    }
  }
} else {
  // 非房主：只與房主建立連線
  await multiP2P.addConnection(ownerUid);
}

// 監聽訊息
multiP2P.onMessage((message) => {
  setMessages(prev => [...prev, message]);
});

// 發送訊息
await multiP2P.sendMessage(message);
```

### 參與者變化處理

```typescript
// 在訂閱回調中
unsubscribeRoomRef.current = RoomService.subscribeRoom(roomId, async (updatedRoom) => {
  const newParticipants = updatedRoom.participants;
  const oldParticipants = lastParticipants;
  
  // 檢測新加入的用戶
  const added = newParticipants.filter(uid => !oldParticipants.includes(uid));
  for (const uid of added) {
    if (isHost && uid !== localUid) {
      await multiP2P.addConnection(uid);
    }
  }
  
  // 檢測離開的用戶
  const removed = oldParticipants.filter(uid => !newParticipants.includes(uid));
  for (const uid of removed) {
    await multiP2P.removeConnection(uid);
  }
  
  lastParticipants = newParticipants;
});
```

## 限制與考量

### 連線數量限制

- **建議最大用戶數**：10-15 人
- **原因**：WebRTC 連線需要資源，過多連線可能導致效能問題

### 效能優化

1. **訊息去重**：使用 messageId 避免重複處理
2. **連線狀態管理**：監控每個連線的狀態
3. **訊息佇列**：如果連線未就緒，將訊息加入佇列
