### 2. 舊房間清理

#### closeAllUserRooms 方法

**檔案**：`src/services/RoomService.ts`

**功能**：
- 關閉用戶的所有房間（包括所有狀態：waiting、open、closed）
- 添加 `closedAt` 時間戳
- 確保用戶不會有多個活躍房間

**實作**：
```typescript
static async closeAllUserRooms(ownerUid: string): Promise<void> {
  // 查詢該用戶的所有房間（不限狀態）
  const q = query(roomsRef, where('ownerUid', '==', ownerUid));
  const snapshot = await getDocs(q);
  
  // 關閉所有房間
  const batch = allRooms.map((r) => {
    const docRef = doc(db, 'p2pRooms', r.roomId);
    return updateDoc(docRef, { 
      status: 'closed',
      closedAt: Timestamp.fromMillis(Date.now()),
    });
  });
  
  await Promise.all(batch);
}
```

**自動調用**：
- 在 `createRoom` 時自動調用
- 確保建立新房間前清理舊房間

### 3. Firestore 安全規則

#### 讀取規則

```javascript
allow read: if isAuthenticated() && (
  // 公開房間
  !resource.data.isPrivate ||
  // 或參與者列表包含當前用戶
  request.auth.uid in resource.data.participants ||
  // 或是房主
  request.auth.uid == resource.data.ownerUid
);
```

#### 建立規則

```javascript
allow create: if isAuthenticated() && 
  request.auth.token.firebase.sign_in_provider != "anonymous" &&
  request.resource.data.ownerUid == request.auth.uid;
```

#### 更新規則

```javascript
allow update: if isAuthenticated() && (
  // 房主可以更新房間
  request.auth.uid == resource.data.ownerUid ||
  // 或參與者可以加入（更新 participants）
  request.auth.uid in request.resource.data.participants
);
```

#### 刪除規則

```javascript
allow delete: if isAuthenticated() && 
  request.auth.uid == resource.data.ownerUid;
```
