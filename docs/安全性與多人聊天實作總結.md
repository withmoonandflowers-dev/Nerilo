#### 1.2 舊房間清理

**修改檔案**：`src/services/RoomService.ts`

**實作內容**：

1. **`closeAllUserRooms` 方法**：
   - 關閉用戶的所有房間（包括所有狀態：waiting、open、closed）
   - 確保用戶建立新房間時，舊房間都被清理
   - 添加 `closedAt` 時間戳

2. **自動清理**：
   - 在 `createRoom` 時自動調用 `closeAllUserRooms`
   - 確保用戶不會有多個活躍房間

**關鍵代碼**：
```typescript
static async closeAllUserRooms(ownerUid: string): Promise<void> {
  // 查詢該用戶的所有房間（不限狀態）
  const q = query(roomsRef, where('ownerUid', '==', ownerUid));
  const snapshot = await getDocs(q);
  
  // 關閉所有房間（包括已經是 closed 狀態的）
  const batch = allRooms.map((r) => {
    const docRef = doc(db, 'p2pRooms', r.roomId);
    return updateDoc(docRef, { 
      status: 'closed',
      closedAt: Timestamp.fromMillis(Date.now()),
    });
  });
  
  await Promise.all(batch);
}
```

#### 1.3 Firestore 安全規則加強

**修改檔案**：`firestore.rules`

**實作內容**：

1. **讀取規則**：
   - 允許讀取公開房間
   - 允許讀取自己參與的房間
   - 允許讀取自己建立的房間

2. **建立規則**：
   - 只允許已登入的用戶（非匿名）建立房間
   - `ownerUid` 必須是當前用戶

3. **更新規則**：
   - 房主可以更新自己的房間
   - 參與者可以加入房間（更新 participants）

4. **刪除規則**：
   - 只允許房主刪除自己的房間

**關鍵規則**：
```javascript
match /p2pRooms/{roomId} {
  // 讀取：公開房間或自己參與的房間
  allow read: if isAuthenticated() && (
    !resource.data.isPrivate ||
    request.auth.uid in resource.data.participants ||
    request.auth.uid == resource.data.ownerUid
  );
  
  // 建立：只允許已登入的用戶（非匿名）
  allow create: if isAuthenticated() && 
    request.auth.token.firebase.sign_in_provider != "anonymous" &&
    request.resource.data.ownerUid == request.auth.uid;
  
  // 更新：房主或參與者
  allow update: if isAuthenticated() && (
    request.auth.uid == resource.data.ownerUid ||
    request.auth.uid in request.resource.data.participants
  );
  
  // 刪除：只允許房主
  allow delete: if isAuthenticated() && 
    request.auth.uid == resource.data.ownerUid;
}
```

### 2. 多人聊天架構設計 ✅

#### 2.1 MultiP2PManager 類別

**檔案**：`src/core/p2p/MultiP2PManager.ts`

**功能**：
- 管理多個 P2P 連線（星型拓撲）
- 房主作為中心節點，其他用戶都與房主建立連線
- 訊息路由和轉發

**關鍵方法**：
- `addConnection(remoteUid)`: 建立與新用戶的連線
- `removeConnection(remoteUid)`: 移除連線
- `sendMessage(message)`: 發送訊息
- `broadcastMessage(message)`: 廣播訊息（房主專用）
- `forwardMessage(fromUid, message)`: 轉發訊息（房主專用）

**架構**：
```
        [房主/Host]
            /  |  \
           /   |   \
    [User1] [User2] [User3]
```
