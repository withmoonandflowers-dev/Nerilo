## 🎯 架構選擇邏輯

系統會根據參與者數量自動選擇架構：

```typescript
const useMesh = room.topology === 'mesh' || effectiveParticipantCount >= 3;
```

- **2 人**：星型拓撲（現有 P2PManager）✅
- **3+ 人**：Mesh 架構（MeshGossipManager）✅
- **topology = 'mesh'**：強制使用 Mesh 架構 ✅

## 🔧 已知限制

### 1. Mesh 連線建立時間
- **時間**：30-60 秒
- **原因**：需要建立多個 P2P 連線、WebRTC signaling、節點發現
- **狀態**：已優化等待時間和超時設置

### 2. 節點發現延遲
- **時間**：2-5 秒
- **原因**：需要等待 Firestore 同步
- **狀態**：已添加重試邏輯（最多 3 次）

### 3. 測試超時
- **問題**：Mesh 測試可能需要更長的超時時間
- **建議**：將超時時間設置為 120 秒

## 📝 使用說明

### 自動啟用
系統會根據參與者數量自動選擇架構，無需手動配置。

### 驗證步驟
1. 打開瀏覽器 Console
2. 查找 `[ChatPage] Deciding P2P architecture` 日誌
3. 確認 `useMesh: true` 時使用 Mesh 架構
4. 查找 `[MeshGossipManager]` 相關日誌
5. 等待 30-60 秒讓連線建立

### 手動測試
1. 打開 3 個瀏覽器
2. 第一個建立房間
3. 第二個和第三個加入
4. 檢查 Console 日誌
5. 等待連線建立
6. 測試訊息傳輸
