# 專案實作邏輯與建議

## 📋 專案架構概述

### 核心技術棧
- **前端框架**：React + TypeScript + Vite
- **後端服務**：Firebase (Auth, Firestore, Functions, Hosting)
- **P2P 通訊**：WebRTC (RTCPeerConnection, RTCDataChannel)
- **測試框架**：Playwright (E2E)

### 主要功能模組

#### 1. 認證與權限管理
- **AuthContext**：管理用戶認證狀態
- **FeatureContext**：基於角色的功能訪問控制 (RBAC)
- **Firebase Auth**：Email/Password、Google 登入
- **Custom Claims**：角色管理（guest, user, admin）

#### 2. 房間管理系統
- **RoomService**：房間 CRUD 操作
- **房間狀態**：waiting → open → closed
- **自動清理**：建立新房間時關閉舊房間
- **權限控制**：只有登入用戶可以建立房間

#### 3. P2P 通訊架構

##### 3.1 星型拓撲（2 人）
```
        [房主/Host]
            /  \
           /    \
      [User1] [User2]
```
- 使用 `P2PManager` 和 `ChatService`
- 房主作為中心節點
- 其他用戶直接與房主連線

##### 3.2 Mesh 架構（3+ 人）
```
    [User1] ←→ [User2]
       ↕          ↕
    [User3] ←→ [User4]
```
- 使用 `MeshGossipManager` 和 `MeshChatService`
- 每個節點維持 k=6 個鄰居連線
- 使用 Gossip 協議傳播訊息
- 訊息簽名驗證防止篡改

#### 4. 訊息處理
- **ChatService**：2 人聊天的訊息處理
- **MeshChatService**：多人聊天的訊息處理
- **IndexedDB**：本地訊息儲存
- **訊息格式**：ChatMessage（標準）、GossipMessage（Mesh）

async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries: number = 3
): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) throw error;

  }

}

class RateLimiter {
  private requests = new Map<string, number[]>();
  
  check(userId: string, maxRequests: number, windowMs: number): boolean {
    const now = Date.now();
    const userRequests = this.requests.get(userId) || [];
    const recent = userRequests.filter(t => now - t < windowMs);
    
    if (recent.length >= maxRequests) {
      return false;
    }
    
    recent.push(now);
    this.requests.set(userId, recent);
    return true;
  }
}
