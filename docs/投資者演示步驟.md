# 投資者演示步驟（可實際操作）

以下為向出資者展示時，**建議的簡短步驟**，每步皆可於瀏覽器中實際操作。

---

## 前置準備（演示前 5 分鐘）

1. **確認環境**  
   - 已部署至 Firebase Hosting，或本機執行 `npm run build` 後 `npm run preview`（或 `npm run dev`）。  
   - Firebase 專案已開啟 **Google 登入**（與必要時 **匿名登入**）。  
   - 準備 **兩種瀏覽情境**：一個已登入（房主）、一個可為無痕/另一瀏覽器（加入者）。

2. **開啟網址**  
   - 部署版：開啟 Hosting 網址（例如 `https://xxx.web.app`）。  
   - 本機：開啟 `http://localhost:4173`（preview）或 `http://localhost:3000`（dev）。

---

## 演示步驟（約 3–5 分鐘）

### 步驟 1：進入首頁

- 開啟上述網址。
- 系統會自動導向 **儀表板**；若尚未登入，會先以**遊客**身份進入（或顯示「載入中…」後進入）。
- 畫面上可見「我的房間」「公開房間」等區塊。

### 步驟 2：登入（才能建立房間）

- 點選右上角 **「登入」**（或畫面中的 **「前往登入」**），進入登入頁。
- 點選 **「使用 Google 登入」**，完成 Google 授權。
- 登入成功後會回到 **儀表板**，右上角會顯示角色（例如 user）與 **「登出」**。

> 若使用 Email/密碼，需先在 Firebase Console 建立該帳號，再於登入頁輸入後點「登入」。

### 步驟 3：建立房間

- 在「我的房間」區塊點 **「+ 建立新房間」**。
- 可輸入房間名稱（選填），再點 **「建立房間」**。
- 建立成功後會自動進入 **等待連線** 頁面，畫面會顯示倒數時間與「分享房間連結」。

### 步驟 4：分享連結並讓第二人加入

- 在等待頁點 **「複製連結」**（或手動複製網址列上的聊天連結）。
- **重要**：第二人請用 **另一瀏覽器** 或 **無痕視窗**（或另一台裝置）貼上連結並開啟；**不要用同一瀏覽器開兩個分頁**（同一帳號無法建立 P2P 連線）。
- 第二人開啟後會自動加入房間；**房主與第二人** 都會自動跳轉到 **聊天室** 頁面。

### 步驟 5：示範即時聊天

- 在聊天室中，任一方在輸入框輸入文字，點 **「傳送」**（或按 Enter）。
- 確認 **雙方畫面皆能即時看到** 對方傳送的訊息。
- 可再示範：連線狀態顯示「已連線」、多則訊息來回、或其中一方點「返回」離開。

## 若無法使用時的檢查

| 狀況 | 可能原因 | 處理方式 |
|------|----------|----------|
| 一直顯示「載入中…」或無法進入儀表板 | Firebase 未設定、網路問題或匿名登入未開啟 | 檢查 Firebase 專案設定與網路；或改為「步驟 2」先登入再操作。 |
| 無法「建立房間」或按鈕為灰色 | 尚未登入（遊客無法建立房間） | 請先完成 **步驟 2** 使用 Google（或 Email）登入。 |
| 複製連結後第二人開啟沒反應 | 連結不完整或貼錯 | 確認複製的是「聊天室」連結（含 `/chat/`），並在另一視窗完整貼上。 |
| 進入聊天室後未顯示「已連線」或無法傳訊 | 第二人用同一瀏覽器/同一帳號開兩分頁，或對方尚未進入聊天室 | 第二人必須用**另一瀏覽器或無痕視窗**開啟連結；雙方都要在聊天室畫面。約 45 秒後畫面會顯示連線說明提示。 |

---

## 一句話版（給簡報用）

「使用者先以 Google 登入 → 建立房間 → 複製連結給另一人 → 兩人進入同一聊天室後即可即時傳訊，全程 P2P、無需伺服器轉傳訊息。」
