## 📐 專案實作邏輯

### 核心架構

#### 雙架構設計

**星型拓撲（2 人）**：
- 使用 `useStarTopology` Hook
- 連線建立快速（5-15 秒）
- 低延遲，直接傳輸
- 連線狀態監聽（事件監聽 + 定期檢查）

**Mesh 架構（3+ 人）**：
- 使用 `useMeshTopology` Hook
- 每個節點維持 k=6 個鄰居連線
- 使用 Gossip 協議傳播訊息
- 連線建立較慢（30-60 秒）

#### 自動架構選擇

```typescript
// 使用 useP2PArchitecture Hook
const decision = architecture.decide(room, effectiveCount);
if (decision.type === 'mesh') {
  await meshTopology.initialize(...);
} else {
  await starTopology.initialize(...);
}
```

### 核心流程

#### 1. 房間建立
```
用戶點擊「建立新房間」
  → 權限檢查（前端）
  → 關閉舊房間（後端）
  → 建立新房間（Firestore）
  → 導向等待頁面
```

#### 2. 加入房間
```
用戶訪問房間 URL
  → 加入房間（Firestore）
  → 檢查房間狀態
  → 訂閱房間變化（useRoomSubscription）
  → 初始化 P2P
```

#### 3. P2P 初始化
```
決定架構（useP2PArchitecture）
  → 初始化對應的拓撲（useStarTopology / useMeshTopology）
  → 建立連線
  → 監聽連線狀態（事件監聽 + 定期檢查）
  → 開始聊天
```

#### 4. 訊息發送

**星型拓撲**：
```
用戶輸入 → starTopology.sendMessage() → ChatService → ChannelBus
```

**Mesh 架構**：
```
用戶輸入 → meshTopology.sendMessage() → GossipHandler → 轉發
```
