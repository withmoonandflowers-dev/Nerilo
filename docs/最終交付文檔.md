## 📐 專案實作邏輯

### 核心架構

#### 雙架構設計

**星型拓撲（2 人）**：
- 使用 `P2PManager` + `ChatService`
- 房主作為中心節點
- 連線建立快速（5-15 秒）

**Mesh 架構（3+ 人）**：
- 使用 `MeshGossipManager` + `MeshChatService`
- 每個節點維持 k=6 個鄰居連線
- 使用 Gossip 協議傳播訊息
- 訊息簽名驗證
- 連線建立較慢（30-60 秒）

#### 自動架構選擇

```typescript
const useMesh = room.topology === 'mesh' || participants.length >= 3;
```

### 核心流程

#### 1. 房間建立
```
用戶點擊「建立新房間」
  → 權限檢查
  → 關閉舊房間
  → 建立新房間
  → 導向等待頁面
```

#### 2. 加入房間
```
用戶訪問房間 URL
  → 加入房間
  → 檢查房間狀態
  → 監聽房間變化
  → 初始化 P2P
```

#### 3. P2P 初始化
```
檢查參與者數量
  → 選擇架構
  → 初始化管理器
  → 建立連線
  → 開始聊天
```

#### 4. 訊息發送

**星型拓撲**：
```
用戶輸入 → ChatService → ChannelBus → RTCDataChannel
```

**Mesh 架構**：
```
用戶輸入 → GossipHandler → 簽名 → 隨機選 2 個鄰居 → 轉發
```
