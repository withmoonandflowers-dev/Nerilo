### 2. 多人聊天架構 ✅

#### 2.1 MultiP2PManager 類別

**檔案**：`src/core/p2p/MultiP2PManager.ts`

**功能**：
- ✅ 管理多個 P2P 連線（星型拓撲）
- ✅ 房主作為中心節點，其他用戶都與房主建立連線
- ✅ 訊息路由和轉發
- ✅ 連線狀態管理
- ✅ 訊息佇列（連線未就緒時）

**架構**：
```
        [房主/Host]
            /  |  \
           /   |   \
    [User1] [User2] [User3]
```

**關鍵方法**：
- `addConnection(remoteUid)`: 建立與新用戶的連線
- `removeConnection(remoteUid)`: 移除連線
- `sendMessage(message)`: 發送訊息
- `broadcastMessage(message)`: 廣播訊息（房主專用）
- `forwardMessage(fromUid, message)`: 轉發訊息（房主專用）
- `onMessage(listener)`: 監聽訊息
- `onConnectionState(remoteUid, listener)`: 監聽連線狀態
- `disconnect()`: 清理所有連線

   - 用戶必須先登入（不能是 guest/匿名用戶）
   - 如果未登入，會自動導航到登入頁面

2. **自動清理**：
   - 建立新房間時，會自動關閉用戶的所有舊房間
   - 確保用戶不會有多個活躍房間

3. **權限檢查**：
   - 前端：UI 按鈕會禁用（guest 用戶）

- 在測試模式下（`--mode test`），允許 guest 用戶建立房間
- 用於 E2E 測試，不影響生產環境的安全性
