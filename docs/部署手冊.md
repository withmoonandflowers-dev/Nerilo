# Nerilo 部署手冊

## 1. 前置準備

### 1.1 環境需求

- Node.js 18+
- npm 或 yarn
- Firebase CLI
- Firebase 專案（已建立）

### 1.2 安裝 Firebase CLI

```bash
npm install -g firebase-tools
firebase login
```

### 1.3 初始化 Firebase 專案

```bash
firebase init
```

選擇：
- Firestore
- Functions
- Hosting

## 2. 環境變數配置

### 2.1 前端環境變數

建立 `.env.local`：

```env
VITE_FIREBASE_API_KEY=your-api-key
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
VITE_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
VITE_FIREBASE_APP_ID=your-app-id
```

### 2.2 Cloud Functions 環境變數

```bash
# 設定 Twilio（可選）
firebase functions:config:set twilio.account_sid="your-account-sid"
firebase functions:config:set twilio.auth_token="your-auth-token"
```

## 3. Firestore 設定

### 3.1 部署 Security Rules

```bash
firebase deploy --only firestore:rules
```

### 3.2 建立索引

```bash
firebase deploy --only firestore:indexes
```

### 3.3 初始化資料

在 Firebase Console 手動建立：

**features 集合範例**：
```json
{
  "featureId": "chat",
  "name": "文字聊天",
  "description": "P2P 文字聊天功能",
  "enabled": true,
  "requiredRoles": ["user", "admin"],
  "route": "/chat/:roomId",
  "icon": "💬",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-01T00:00:00Z"
}
```

## 4. Cloud Functions 部署

### 4.1 建置 Functions

```bash
cd functions
npm install
npm run build
```

### 4.2 部署 Functions

```bash
firebase deploy --only functions
```

### 4.3 設定使用者角色（首次部署後）

透過 Firebase Console 或 Cloud Functions：

```typescript
// 在 Firebase Console > Functions > setRole
// 或使用 Firebase CLI
firebase functions:shell
> setRole({ uid: 'user-uid', role: 'admin' })
```

## 5. 前端建置與部署

### 5.1 安裝依賴

```bash
npm install
```

### 5.2 建置專案

```bash
npm run build
```

### 5.3 部署到 Firebase Hosting

```bash
firebase deploy --only hosting
```

## 6. 開發環境設定

### 6.1 使用 Firebase Emulator

```bash
# 安裝依賴
npm install

# 啟動 Emulator
firebase emulators:start
```

### 6.2 前端開發伺服器

```bash
npm run dev
```

前端會連接到 Emulator（需設定環境變數）。

## 7. 生產環境檢查清單

### 7.1 安全性檢查

- [ ] Firestore Rules 已部署
- [ ] 環境變數已設定（不包含敏感資訊）
- [ ] Twilio credentials 僅在 Functions（不在前端）
- [ ] Custom Claims 已正確設定

### 7.2 功能檢查

- [ ] 登入/登出正常
- [ ] Feature Registry 正常載入
- [ ] P2P 連線可建立
- [ ] 文字訊息可收發
- [ ] 檔案傳輸可用
- [ ] 影音通話可用

### 7.3 資料隱私檢查

- [ ] Firestore 中無聊天文字
- [ ] Firestore 中無檔案本體
- [ ] Firestore 中無影音內容
- [ ] 僅 signaling 資料在 Firestore

## 8. 監控與維護

### 8.1 監控指標

### 8.3 錯誤處理

## 9. 備份與還原

### 9.1 Firestore 備份

```bash
# 使用 Firebase CLI 匯出
gcloud firestore export gs://your-bucket/backup
```

### 9.2 還原

```bash
gcloud firestore import gs://your-bucket/backup
```

## 10. 效能優化

### 10.1 Firestore 索引

確保常用查詢已建立索引（見 `firestore.indexes.json`）。

### 10.2 Functions 優化

- 使用適當的記憶體配置
- 設定 timeout
- 使用快取（如適用）

### 10.3 前端優化

- 程式碼分割
- 懶加載路由
- 圖片優化

## 11. 故障排除

### 11.1 P2P 連線失敗

- 檢查 ICE servers 配置
- 檢查防火牆/NAT 設定
- 確認 TURN server 可用（如使用）

### 11.2 Firestore Rules 拒絕

- 檢查 Custom Claims
- 檢查 Rules 邏輯
- 查看 Functions 日誌

### 11.3 功能未顯示

- 檢查 features 文件 enabled=true
- 檢查 requiredRoles
- 檢查使用者 role

## 12. 更新流程

### 12.1 前端更新

```bash
npm run build
firebase deploy --only hosting
```

### 12.2 Functions 更新

```bash
cd functions
npm run build
firebase deploy --only functions
```

### 12.3 Rules 更新

```bash
firebase deploy --only firestore:rules
```

## 13. 回滾程序

### 13.1 前端回滾

在 Firebase Console > Hosting > Releases 選擇舊版本回滾。

### 13.2 Functions 回滾

```bash
firebase functions:delete functionName
firebase deploy --only functions:functionName@previous-version
```

## 14. 安全最佳實踐

1. **定期更新依賴**：`npm audit fix`
2. **監控異常活動**：Firebase Console > Usage
3. **限制 Functions 存取**：使用適當的 IAM 權限
4. **定期審查 Rules**：確保符合安全需求
5. **使用環境變數**：不將敏感資訊寫入程式碼
