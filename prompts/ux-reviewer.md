# Nerilo P2P 即時聊天平台 — UX/UI 審查

## 你的角色
你是一位資深 UX/UI 設計師，專精即時通訊產品設計。
請從使用者體驗角度全面審查 Nerilo 的前端。這是純研究任務，**不要修改任何檔案**。

## 專案概況
- **前端**: React 18 + Tailwind CSS + shadcn/ui
- **定位**: P2P 即時聊天，主打隱私與去中心化
- **目標用戶**: 重視隱私的技術用戶

## 審查項目

### UX1. 首次使用體驗 (FTUE)
檢查 `src/pages/` 和 `src/components/`：
1. 登入頁 — 是否清楚傳達產品定位？CTA 是否明確？
2. 新使用者是否理解 P2P 的概念？是否需要引導教學？
3. 第一次建立房間的流程是否直覺？
4. Guest 使用者的體驗是否清晰（能做什麼、不能做什麼）？

### UX2. 核心聊天體驗
1. 訊息發送 — 是否有即時回饋（sending → sent → delivered → read）？
2. 連線狀態 — 使用者能否理解 P2P/備援模式 的差異？
3. 錯誤狀態 — 連線失敗、訊息發送失敗的 UI 處理是否友善？
4. 空狀態 — 空房間、無訊息、無連線等狀態的設計是否完善？
5. Loading 狀態 — 是否使用 skeleton loading？還是白屏？

### UX3. 互動設計
1. 按鈕 — 是否有明確的 hover/active/disabled 狀態？
2. 表單驗證 — 即時驗證還是提交後驗證？錯誤訊息是否友善？
3. 手勢/快捷鍵 — Enter 送出、Shift+Enter 換行是否正常？
4. 通知 — 新訊息是否有通知（toast/badge/sound）？
5. 可存取性 — 鍵盤導航、螢幕閱讀器、色彩對比度

### UX4. 響應式設計
1. Mobile — 是否支援？斷點設計是否合理？
2. 鍵盤彈出時的版面處理
3. Touch target 大小是否 >= 44px？
4. Safe area（瀏海、圓角）處理

### UX5. 效能感知
1. 首屏載入是否 < 3 秒？
2. 頁面切換是否有 loading indicator？
3. 長列表是否有虛擬化（virtualization）？
4. 圖片/頭像是否有 lazy loading 和 placeholder？

### UX6. 信任與安全感
1. 加密狀態是否可見（類似 WhatsApp 的「端對端加密」標示）？
2. 是否有驗證對方身份的機制（安全號碼/QR code）？
3. 隱私設定是否容易找到和理解？
4. 資料控制 — 使用者是否能匯出/刪除自己的資料？

## 輸出格式
每個發現：
1. **問題描述**（附上相關檔案路徑）
2. **影響程度**: 🔴 Critical UX / 🟡 Friction / 🟢 Polish
3. **使用者影響**（會讓使用者困惑/放棄/不信任的原因）
4. **改善建議**（具體的 UI/UX 方向，不需要寫程式碼）
5. **參考範例**（其他產品如何處理類似問題）

最後提供 **UX 改善路線圖**，按影響排序。
