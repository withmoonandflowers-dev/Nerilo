# Spec 001：為盲信使寄存側建立經濟均衡

- 軌別：feature（quote/receipt 格式已由 Protocol Spec 003 v1 固定）
- 狀態：done
- 建立：2026-07-14／最後更新：2026-07-16（T0–T6、V1–V5 全數完成）
- 關聯：ADR-0024（盲信使儲存經濟學）、ADR-0022（可驗證點數帳本）、風險登記 R5、**依賴 Spec 002（R5 收斂，前置）**

## 1. 要做什麼、為什麼（specify）

盲信使目前寄存免費，且寄存動作本身會刷新 LRU 熱度（CourierStore lastAccessedAt），攻擊者能以趨近零的成本持續寄存垃圾密文並保溫自己的房，把誠實房間擠出信使快取。因為信使是盲的（看不懂密文），內容過濾天生不可行，防線只能建立在出處與經濟上。本規格以價格理論建立寄存側經濟均衡：供給隨在線人數內生、需求付市價、每筆債權都有明確對象、貢獻取得可交換欠條，使垃圾寄存從「零成本傷害他人」變成「消耗自己在該信使的信用額度」，外部性內部化。

**憲法檢核**（constitution.md）：
- 目標函數加分項：隱私韌性（信使網路是斷網送達的地基）；補助競爭力（分散式價格均衡設計是技術深度敘事的一章）。
- 四條不變量影響：恰好一次〈無〉；E2EE〈無〉；帳本正當性〈有，依賴 Spec 002 先收 R5〉；身分授權〈有，per-身分配額綁非匿名反女巫〉。

## 2. 邊界（明確不做）

- 不做全域共識帳本／區塊鏈（維持 ADR-0011 觸發條件判準）。
- 不做 coin、全網餘額或鑄造銷毀制貨幣；採有明確發票人與持有人的欠條，債權只在交換時移轉。
- 不做協議層中央定價；協議只定義市場原語（quote/receipt），定價留各節點自主。
- 不做檔案／媒體代存配額市場（ADR-0024 延後）。
- 不做內容審查（盲性是特性）。
- LRU 大修（淘汰權重乘貢獻分）不做，留觀察；只做小修（寄存不刷熱度）。

## 3. 待釐清（clarify）——已全數拍板 2026-07-14

- [x] Q1 計價單位：**byte×天**（storage）與 **byte**（serve/relay）。價格理論模型見第 4 節；使用者要求以供需均衡而非固定費。
- [x] Q2 冷啟動：**生產者優先**。新身分在每個信使各有未結欠條上限 ε（夠輕度使用、不夠攻擊）；貢獻取得其他使用者開的服務欠條，經三方同意交換後，把自己的欠條取回並由本人結清，恢復該信使額度。
- [x] Q3 執行位置：**本地裁量定價 + 協議層定原語**（quote 報價、receipt 收據、拒收語義）。定價演算法不入協議。
- [x] Q4 LRU：**只做小修**——寄存不刷新 lastAccessedAt，僅成員 pull（真實使用）刷新。大修不做。

## 4. 技術計畫（plan）

### 4.1 經濟模型（分散式擁擠定價，有對象欠條）

市場資源：寄存儲存（byte×天）、補齊頻寬（byte）、中繼轉發（byte）、訊號引導（次）。
總供給 S ≈ N（在線節點數）× 人均預算 × 在線率，隨參與人數內生；總需求 D 隨活躍房間與訊息量走。

個體定價（每信使自主，局部調價）：

```
p_j(t+1) = clamp( p_j(t) × [1 + λ(u_j(t) − u*)] , p_min , p_max )
u_j = 占用/預算；u* = 目標使用率（初值 0.7）；λ = 調整速度（初值 0.1/輪）
```

消費者選擇（寄存 q byte·天、冗餘 k 份）：

```
min Σ_{j∈K} p_j·q   s.t. |K| = k，對每個信使 j 的未結欠條 d_ij + p_j·q ≤ ε_ij
```

套利使活躍信使價格趨同（一價定律）；市場出清 Z(p*) = D(p*) − S(p*) = 0。
需求端天然價格上限 = 「成員互補（免費但慢）」的出走選擇權，市場不失控且免費底線服務保留（公共財性質）。

欠條面（2026-07-16 實作期修訂）：

```
寄存：使用者 i 向信使 j 開欠條 IOU(i→j, p_j·q)，d_ij 增加
貢獻：受益人 a 向服務者 i 開欠條 IOU(a→i, w·服務量)
交換：IOU(a→i) 經 a（原發票人）+ i（目前持有人）+ j（新持有人）同意，移轉給 j
結清：j 把 IOU(i→j) 退回 i；只有發票人 i 本人簽結清，d_ij 才減少
```

這不是 coin：不同發票人的欠條不是同一資產，只在交易雙方同意的價格下交換。每個信使只對自己持有的債權作權威判斷，不需全域餘額或共識。冷啟動 `d_ij(0)=0`、`d_ij≤ε_ij`；ε 綁簽章身分，女巫成本 = 開分身成本 × ε。

### 4.2 落地切面

- `CourierStore`：寄存不再刷新 lastAccessedAt（Q4 小修）；新增 per-寄存者（簽章身分）占用統計。
- `CourierService`：DEPOSIT 前取得 QUOTE，寄存者按報價簽發本人欠條；信使驗身分、簽章、報價與 per-發票人未結上限，超額拒收。
- `CourierIOUBook` + `CoSignedReceipt`：信使維護自己持有的寄存債權；貢獻收據代表「受益人欠服務者」，三方簽署轉讓後可交換；自己的欠條只有本人簽署才結清。
- 定價器：純函式模組（利於單元測試與模擬），信使側掛載。
- 協議原語標記：QUOTE 與 receipt 的線上格式，完成後開 protocol 軌 spec 記錄（供未來多實作互通）。

### 4.3 取捨（完成後回填 ADR）

- coin vs 有對象欠條：選欠條；沒有全網同質餘額，不需央行或共識。不同欠條的交換價格由交易雙方同意。
- 局部擁擠定價 vs 全域喊價：去中心化無喊價者，局部調價 + 套利收斂是可實作的等價物。
- ε 初始配額 vs 純零起步：純零起步會把「只想跟朋友聊天的輕使用者」也擋在門外，違反公共財定位。
