# Spec 001：為盲信使寄存側建立經濟均衡

- 軌別：feature（含 protocol 成分：收據與計價語義最終應進協議軌）
- 狀態：clarifying
- 建立：2026-07-14／最後更新：2026-07-14
- 關聯：ADR-0024（盲信使儲存經濟學）、ADR-0022（可驗證點數帳本）、風險登記 R5

## 1. 要做什麼、為什麼（specify）

盲信使目前寄存免費，且寄存動作本身會刷新 LRU 熱度（CourierStore lastAccessedAt），攻擊者能以趨近零的成本持續寄存垃圾密文並保溫自己的房，把誠實房間擠出信使快取。因為信使是盲的（看不懂密文），內容過濾天生不可行，防線只能建立在出處與經濟上。本規格把「寄存」從免費改為受經濟約束：讓攻擊成本正比於其占用的資源，讓誠實貢獻者獲得優先服務，使垃圾寄存在經濟上不划算。

**憲法檢核**（constitution.md）：
- 目標函數加分項：隱私韌性（信使網路是斷網送達的地基，被污染等於韌性失效）；補助競爭力（經濟均衡設計是技術深度敘事的一章）。
- 四條不變量影響：恰好一次〈無直接影響——寄存層在對帳之下〉；E2EE〈無——不碰加密〉；**帳本正當性〈有——經濟迴路閉合後，點數的花費側首次出現，R5（earn 不強制收據）成為地基問題必須先收〉**；身分授權〈有——per-寄存者配額需要穩定的簽章身分〉。

## 2. 邊界（明確不做）

- 不做全域共識帳本／區塊鏈（維持 ADR-0011 的觸發條件判準）。
- 不做檔案／媒體代存的配額市場（ADR-0024 已明確延後）。
- 不做內容審查——盲性是特性，防線只在出處與經濟。
- 不改共簽收據的密碼學格式（沿用 CoSignedReceipt，只擴大其用途）。
