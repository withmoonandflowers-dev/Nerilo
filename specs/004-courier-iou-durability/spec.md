# Spec 004：信使欠條簿本機耐久與復原

- 狀態：completed
- 日期：2026-07-16
- 關聯：ADR-0029、ADR-0030、Protocol Spec 003

## 目的

讓每個信使的獨立欠條簿在同一瀏覽器重載後保留未結額度、已接受服務欠條與防重放狀態，同時避免損壞本機資料被當成合法債權。

## 驗收

- [x] D1：以信使 `ownerNodeId` 隔離快照，保存寄存債權、issuer 公鑰、已接受貢獻與定價狀態；不保存短命報價。
- [x] D2：重載逐張重驗身分綁定、簽章、holder 與 per-issuer ε；壞資料被移除且不拖垮整本恢復。
- [x] D3：寄存紀錄與債權都耐久後才 ACK；任一保存失敗回滾兩邊並回 `persistence-failed`。
- [x] D4：結清與防重放狀態以單一快照原子保存；寫入失敗恢復舊債權。
- [x] D5：單元測試涵蓋重載、損壞快照、保存失敗與 receipt replay；Vue emulator E2E 涵蓋信使重載後紀錄及未結欠條仍在。

## 不在本規格

跨裝置備份、私鑰匯出／復原與多副本衝突合併。這三者必須一起設計，否則複製欠條 JSON 既不能證明信使身分，也可能造成雙重狀態。
