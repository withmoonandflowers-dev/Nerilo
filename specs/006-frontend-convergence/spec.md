# Spec 006：前端收斂 — 畫面單純、聚焦

- 軌別：feature
- 狀態：clarifying
- 建立：2026-07-16／最後更新：2026-07-16
- 關聯：ADR-0017（Vue/Nuxt 接班）、**依賴 Spec 005**（連線模型定案後，UI 才好定案）

## 1. 要做什麼、為什麼（specify）

使用者操作前端時覺得「好亂」，要收斂功能、畫面單純。目前 dashboard 同時塞了：我的房間、公開房間、功能搜尋、遊戲入口、調色盤等，資訊密度高。使用者提到「有可以玩遊戲的版本比較好」，但核心訴求是**收斂**：把畫面砍到只留最重要的路徑，降低認知負擔。本 spec 決定收斂後的資訊架構與保留範圍。

**憲法檢核**：
- 目標函數加分項：可維運（少即是好維護）；補助競爭力（clean demo 比雜亂 demo 有說服力）。
- 四條不變量影響：四條皆〈無〉（純前端資訊架構，不動核心）。

## 2. 邊界（初步）

- 收斂在 **Vue/Nuxt 接班版**（`web-vue/`）上做；React 生產版凍結不動（ADR-0017）。
- 不動核心傳輸/加密；只動畫面與導覽。
- 不新增功能；本 spec 是「砍與合」，不是「加」。

## 3. 待釐清（clarify）——等你拍板

- [ ] Q1 **保留範圍**：收斂後留哪些？
  - (a) 只留聊天（最單純）
  - (b) 聊天 + 遊戲（你說遊戲版本比較好）
  - (c) 聊天為主、遊戲當房內的一個可選面板
- [ ] Q2 **「遊戲版本比較好」指什麼**：是某個你記得的舊介面風格（要我找出來參考）？還是「以遊戲為進入點」的產品方向？
- [ ] Q3 **首頁砍什麼**：目前 dashboard 有 我的房間／公開房間／功能搜尋／遊戲／調色盤。哪些是雜訊要砍或收起？（例：公開房間目錄、功能搜尋框）
- [ ] Q4 **進入模型**：收斂後主路徑是「建房間→傳連結」還是「一鍵找人連上就聊」？（這與 Spec 005 的 bootstrap 方式綁在一起）
- [ ] Q5 **調色盤/主題**：保留多主題切換，還是砍到單一乾淨主題？
